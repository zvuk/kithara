## Bug A: `loaded_segment_offset_mismatch` regression

### Hypothesis A1: isolated segment with DRM drift is wrongly classified as mismatch

**Code:** `downloader.rs:357-381`
**Introduced by:** commit `b3a326d9f`

The function returns `Some((loaded, expected))` for any isolated segment whose offset
differs from `expected_layout_offset`. But for DRM re-downloads, the loaded offset
IS the correct reconciled offset — it just differs from the pre-reconciliation layout.

**Fix approach:** When a segment has no contiguous neighbors AND belongs to the
committed variant without midstream switch, trust the loaded offset.

### Status: FIXED ✅
Removed `loaded_segment_offset_mismatch` guard from `resolve_byte_offset`.
All 115 kithara-hls unit tests pass.

## Bug B: DRM SeekFailed("unexpected end of file")

### Key observation from test log:
```
wait_range: EOF range_start=25337253 total_bytes=25337253 num_entries=37
seek anchor path: exact decoder seek failed err=SeekFailed("unexpected end of file") epoch=11
```

All 37 segments loaded, but range_start == total_bytes → EOF.
Seek to 205.81s (93.5% of 220s track) resolves to the LAST byte position.
This means total_bytes is too small — the stream data extends beyond total_bytes.

### Hypothesis B1: cascade_contiguity shifts segments but doesn't update total_bytes

When `cascade_contiguity` relocates segments, it changes byte_offsets but NOT total_bytes.
If the last segment is relocated to a smaller offset, its end_offset shrinks.
But total_bytes stays at the reconciled sum, which may differ from the actual layout.

Result: gap at end of stream, positions within gap are unreachable.

### Hypothesis B1: byte_len not updated after ABR switch (base_offset > 0)
- Fixed `update_decoder_len_for_seek` to compute relative length
- Result: STILL FAILS
- `range_start=25337253 == total_bytes=25337253` — Symphonia seeks to exact end
- byte_len update is NOT the issue — the stream position itself is wrong

### Hypothesis B2: total_bytes undercount from reconciliation drift
The actual stream data may extend past `total_bytes`. 37 DRM segments reconciled,
each might subtract slightly more than actual padding → cumulative error.
If `total_bytes` is even 1 byte less than actual last segment end, the last
byte of real data is past EOF boundary.
